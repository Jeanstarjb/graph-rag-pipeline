# ⚔️ Attack on Titan — Hybrid GraphRAG Pipeline

> A production-grade, multi-database Retrieval-Augmented Generation (RAG) system that answers deep lore questions about *Attack on Titan* by combining a **Neo4j Knowledge Graph** and a **ChromaDB Vector Store**, orchestrated by an intelligent **LangGraph agent**.

---

## 🏗️ Architecture

The system is built in three distinct stages:

```
┌─────────────────────────────────────────────────────────────┐
│                        DATA INGESTION                       │
│          Fandom Wiki ──► cloudscraper ──► raw_lore.txt      │
└────────────────────────────┬────────────────────────────────┘
                             │
             ┌───────────────┴───────────────┐
             ▼                               ▼
┌────────────────────┐           ┌─────────────────────────┐
│   VECTOR DATABASE  │           │    KNOWLEDGE GRAPH      │
│   ChromaDB (local) │           │    Neo4j AuraDB (cloud) │
│   BAAI/bge-small   │           │    gpt-4o-mini extracts │
│   49 chunks        │           │    nodes & relationships│
└────────────────────┘           └─────────────────────────┘
             │                               │
             └───────────────┬───────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                     LANGGRAPH AGENT                         │
│  [Router] ──► [Graph Retriever] ──► [Generator] ──► Answer │
│          └──► [Vector Retriever] ──►               │
└─────────────────────────────────────────────────────────────┘
```

### Agent Routing Logic

The LangGraph agent uses `llama-3.1-8b-instant` (Groq) to intelligently classify each question:

| Route | Triggered for | Retrieval Method |
|---|---|---|
| `graph` | Relational questions (*"Who killed X?"*, *"Who allied with Y?"*) | Cypher query via `GraphCypherQAChain` |
| `vector` | Descriptive questions (*"What does the Colossal Titan look like?"*) | Semantic similarity search |
| `both` | Ambiguous or multi-faceted questions | Parallel retrieval from both sources |

## 🚀 Quick Start

### 1. Prerequisites

- Python 3.10+
- A [Groq API Key](https://console.groq.com/) (free)
- A [Neo4j AuraDB Free](https://neo4j.com/cloud/aura-free/) instance
- A [GitHub Personal Access Token](https://github.com/settings/tokens) **or** an OpenAI API Key (for graph extraction only)

### 2. Install Dependencies

```bash
pip install -r requirements.txt
pip install langchain-openai cloudscraper langchain-text-splitters
```

### 3. Configure Environment Variables

Copy the example file and fill in your credentials:

```bash
cp .env.example .env
```

Edit `.env`:

```env
# Groq — Used by the live query agent
GROQ_API_KEY="gsk_..."

# For graph extraction only — use one of:
GITHUB_TOKEN="github_pat_..."   # Preferred (free via github.com/marketplace/models)
# or
OPENAI_API_KEY="sk-..."

# Neo4j AuraDB
NEO4J_URI="neo4j+s://<your-id>.databases.neo4j.io"
NEO4J_USERNAME="neo4j"
NEO4J_PASSWORD="your-aura-password"
```

### 4. Run the Commander

```bash
python main.py
```

You'll see the interactive menu:

```
==================================================
HYBRID GRAPHRAG COMMANDER (ATTACK ON TITAN LORE)
==================================================
1. Run the Scraper (Harvest Lore)
2. Build the Databases (Neo4j & ChromaDB)
3. Chat with the Hybrid GraphRAG Agent
4. Exit
==================================================
```

**First run:** Execute steps in order: `1` → `2` → `3`.

## 🧠 Technology Stack

| Layer | Technology | Role |
|---|---|---|
| Scraping | `cloudscraper` + `BeautifulSoup4` | Bypass Cloudflare, extract wiki text |
| Graph Extraction | `langchain-experimental` `LLMGraphTransformer` | Convert text to nodes & edges |
| Graph LLM | `gpt-4o-mini` via GitHub Models API | Accurate JSON schema generation |
| Knowledge Graph | Neo4j AuraDB | Cloud graph database |
| Embeddings | `BAAI/bge-small-en-v1.5` (HuggingFace) | Local embedding model |
| Vector Store | ChromaDB | Local persistent vector database |
| Agent LLM | `llama-3.1-8b-instant` via Groq | Fast, free inference for routing & generation |
| Orchestration | LangGraph | Stateful multi-node agent workflow |

## 🗺️ Knowledge Graph Schema

The Neo4j graph contains entities and relationships extracted from the wiki lore:

**Node Types:** `Person`, `Titan`, `Location`, `Faction`, `Event`, `Concept`

**Relationship Types:** `KILLED`, `ALLIED_WITH`, `POSSESSES`, `LOCATED_IN`, `PART_OF`, `PARTICIPATED_IN`

---

## ⚠️ Notes

- **Neo4j Database Name**: The free AuraDB tier assigns a unique hex ID (e.g. `93da935a`) instead of the default `neo4j`. This is hardcoded in `src/knowledge_graph.py` and `src/agent.py` — update if you recreate your instance.
- **Graph Extraction Model**: Small open-source LLMs (7B–70B) tend to hallucinate the JSON schema required by `LLMGraphTransformer`. The pipeline uses `gpt-4o-mini` (via GitHub Models, which is free up to rate limits) for this step only.
- **Rate Limits**: The graph extraction step calls the LLM once per text chunk (49 calls). You may hit GitHub Models rate limits for large re-builds.
